**Endpoint:** `GET /v1/realtime` (WebSocket upgrade)

The Realtime API enables low-latency, multimodal interactions over WebSocket. It supports voice conversations, text chat, and function calling in real time.

## Authentication

The gateway reads the API key from the WebSocket subprotocol header:

```
Sec-WebSocket-Protocol: realtime, openai-insecure-api-key.YOUR_API_KEY, openai-beta.realtime-v1
```

Alternatively, set `Authorization: Bearer YOUR_API_KEY` before upgrade if your client supports it.

## Connection

```
{{ws-base-url}}/v1/realtime?model=gpt-4o-realtime-preview
```

## Usage with OpenAI SDK

```python
from openai import OpenAI

client = OpenAI(
    base_url="{{api-base-url}}",
    api_key="YOUR_API_KEY",
)

# Connect to Realtime API
# The SDK manages WebSocket connection and event handling
```

## Supported Features

| Feature | Description |
| --- | --- |
| **Voice conversations** | Real-time speech-to-speech interaction |
| **Text chat** | Send and receive text messages |
| **Function calling** | Model can call your functions in real time |
| **Session configuration** | Set instructions, voice, modalities, and tools |
| **Audio input/output** | PCM16 audio at configurable sample rates |
| **Interruption** | Cancel in-progress model responses |

## Session Configuration

```json
{
  "type": "session.update",
  "session": {
    "modalities": ["text", "audio"],
    "instructions": "You are a helpful assistant.",
    "voice": "alloy",
    "input_audio_format": "pcm16",
    "output_audio_format": "pcm16",
    "turn_detection": {
      "type": "server_vad",
      "threshold": 0.5,
      "prefix_padding_ms": 300,
      "silence_duration_ms": 500
    }
  }
}
```

<Info>

Requires an upstream channel that supports OpenAI Realtime for the requested model.
