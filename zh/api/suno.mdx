通过 Starrise AI 网关创建 AI 生成的音乐。任务为异步模式 — 提交生成请求后轮询获取结果。

**认证:** `Authorization: Bearer YOUR_API_KEY`

## 提交音乐生成

```bash
curl {{api-origin}}/suno/submit/generate \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "充满活力的电子舞曲，带有动感合成器",
    "mv": "chirp-v4"
  }'
```

路径中的 `:action` 对应上游 Suno 任务类型：

| Action | 说明 |
| --- | --- |
| `generate` | 标准音乐生成 |
| `custom_generate` | 自定义歌词生成 |
| `extend` | 扩展现有音频 |
| `generate_lyrics` | 仅生成歌词 |

## 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `prompt` | string | 是 | 音乐风格和描述 |
| `mv` | string | 否 | 模型版本（如 `chirp-v4`） |
| `title` | string | 否 | 歌曲标题 |
| `tags` | string | 否 | 音乐流派标签 |
| `continue_at` | number | 否 | 从指定秒数开始扩展 |
| `continue_clip_id` | string | 否 | 待扩展的音频片段 ID |
| `make_instrumental` | boolean | 否 | 仅生成纯音乐 |
| `gpt_description_prompt` | object | 否 | 自定义歌词生成提示 |

## 查询单个任务

```bash
curl {{api-origin}}/suno/fetch/{task_id} \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## 批量查询

```bash
curl {{api-origin}}/suno/fetch \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "ids": ["task-id-1", "task-id-2"]
  }'
```

## 任务状态

任务为异步模式 — 轮询查询接口直至状态显示完成。常见状态包括 `pending`、`processing`、`completed`、`failed`。

计费记录与其他转发调用一样显示在控制台使用日志中。
