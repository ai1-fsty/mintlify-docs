## 向量化

**端点:** `POST /v1/embeddings`

将文本转换为向量表示，用于语义搜索、聚类、推荐和检索增强生成（RAG）。

```bash
curl {{api-base-url}}/embeddings \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "text-embedding-3-small",
    "input": "这家餐厅的食物美味，服务也很出色。"
  }'
```

### 响应

```json
{
  "object": "list",
  "data": [
    {
      "object": "embedding",
      "index": 0,
      "embedding": [0.01, 0.02, ...]
    }
  ],
  "model": "text-embedding-3-small",
  "usage": {
    "prompt_tokens": 12,
    "total_tokens": 12
  }
}
```

### Python SDK

```python
from openai import OpenAI

client = OpenAI(
    base_url="{{api-base-url}}",
    api_key="YOUR_API_KEY",
)

response = client.embeddings.create(
    model="text-embedding-3-small",
    input=["文档一", "文档二"],
    dimensions=512
)
print(len(response.data[0].embedding))
```

### 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `model` | string | 是 | 向量化模型 ID |
| `input` | string/array | 是 | 待向量化的文本（字符串或字符串数组） |
| `dimensions` | integer | 否 | 输出维度（取决于模型） |
| `encoding_format` | string | 否 | `float` 或 `base64` |

## 重排序

**端点:** `POST /v1/rerank`

Jina 兼容的重排序接口，用于提升检索质量。按与查询的相关性对文档重新排序：

```bash
curl {{api-base-url}}/rerank \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "jina-reranker-v2-base-multilingual",
    "query": "法国的首都是哪里？",
    "documents": [
      "巴黎是法国的首都。",
      "柏林是德国的首都。",
      "伦敦是英国的首都。"
    ],
    "top_n": 3
  }'
```

### 参数

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `model` | string | 是 | 重排序模型 ID |
| `query` | string | 是 | 搜索查询 |
| `documents` | array | 是 | 文档字符串列表 |
| `top_n` | integer | 否 | 返回的 Top N 结果数量 |

### 应用场景

- **语义搜索**: 向量化查询和文档，查找最近邻
- **RAG**: 检索相关文档，重排序提升质量，送入大模型
- **聚类**: 按向量距离分组相似文档
- **推荐**: 通过向量距离查找相似内容
